# Session Learning Skill

セッション中の会話（ユーザーからの指摘、議論、最終決定）から知識を抽出し、ガイドラインスキルに蓄積するスキルです。

## 使用方法

```
/session-learning
```

議論が一段落した後、学習させたい内容がある場合に実行します。

---

## 重要なルール: 機密情報の保護

ガイドラインスキルに追記する際は、**会社の情報を含めない**こと。

### 含めてはいけない情報
- プロジェクト名、リポジトリ名、組織名
- 具体的なファイルパス、コンポーネント名
- 社内用語、プロジェクト固有の命名
- ユーザー名、チーム名

### 記載方法
- **学習元**: `セッション内議論 (yyyy-mm)` のように抽象化
- **コード例**: 一般的な命名に置き換えてぼかす
- **内容**: 具体的な業務内容ではなく、技術的な知見として一般化

---

## 実行手順

### 1. 会話の分析

現在のセッションの会話を分析し、以下のパターンを探す：

#### ユーザーからの指摘
- 「こうすべき」「これは違う」「〜した方がいい」
- 問題点の指摘、修正要求
- コードや設計への改善提案

#### 議論の経緯
- 検討した選択肢とそのトレードオフ
- なぜその方法が良いか/悪いかの理由
- 代替案の検討

#### 最終結論
- 採用した設計・実装方針
- 最終的なコードの形
- 合意に至った理由

### 2. 学習観点での抽出

以下の観点で知識を抽出する：

#### コーディングパターン
- 採用すべき実装パターン
- 避けるべきアンチパターン
- 命名規則・コードスタイル

#### 設計判断
- アーキテクチャ上の決定とその理由
- トレードオフの考慮
- コンポーネント分割の基準

#### ベストプラクティス
- ユーザーが重視するポイント
- プロジェクト固有の慣習（一般化して記載）
- 品質基準

### 3. 該当ガイドラインスキルの特定

抽出した知識を以下のスキルに分類：

| 知識の種類 | 保存先 |
|-----------|--------|
| コンポーネント設計、単一責任、props/emit、レイアウト | `~/dotfiles/.claude/skills/guideline-component-design/SKILL.md` |
| Vue/Nuxt固有、useState、computed、SSR | `~/dotfiles/.claude/skills/guideline-vue-nuxt/SKILL.md` |
| 日時処理、タイムゾーン | `~/dotfiles/.claude/skills/guideline-datetime/SKILL.md` |
| ライブラリ選定・活用 | `~/dotfiles/.claude/skills/guideline-library-usage/SKILL.md` |
| その他（新規カテゴリ） | 新規スキル作成を提案 |

### 4. ガイドラインスキルへの追記

該当する`~/dotfiles/.claude/skills/guideline-*/SKILL.md`の「学習した知識」セクションに追記：

```markdown
### [学習内容のタイトル]
**学習元**: セッション内議論 (yyyy-mm)

**内容**:
[具体的な学習内容（一般化して記載）]

**コード例**:
```[言語]
[一般的な命名に置き換えたコード]
```

**理由・背景**:
[なぜこのパターン/方針が良いのか]
```

### 5. 追加内容の確認

追記内容をユーザーに提示し、確認を求める。必要に応じて修正。

## 出力形式

### 学習レポート
```markdown
# セッション学習レポート

## 抽出した知識

### 1. [知識タイトル1]
- **カテゴリ**: guideline-component-design
- **概要**: [要約]
- **追記内容**: [ガイドラインに追加する内容（一般化）]

### 2. [知識タイトル2]
- **カテゴリ**: guideline-vue-nuxt
- **概要**: [要約]
- **追記内容**: [ガイドラインに追加する内容（一般化）]

## アクション
上記の内容を該当するガイドラインスキルに追記しますか？
```

## 新規ガイドラインスキルの作成

既存のガイドラインに該当しない知識を発見した場合：

1. ユーザーに新規スキル作成を提案
2. 承認されたら `~/dotfiles/.claude/skills/guideline-[カテゴリ名]/SKILL.md` を以下の形式で作成：

```markdown
---
name: guideline-[カテゴリ名]
description: [説明。実装時に自動参照されるようキーワードを含める]
---

# [カテゴリ名]ガイドライン

## 基本原則
[議論から抽出した原則]

## アンチパターン
[避けるべきパターン]

## 推奨パターン
[推奨されるパターン]

---

## 学習した知識

### [最初の学習内容]
**学習元**: セッション内議論 (yyyy-mm)
...
```

## 学習サイクル

```
[コーディング中にユーザーから指摘を受ける]
    ↓
[議論を経て設計・実装方針が決定]
    ↓
[/session-learning を実行]
    ↓
[知識を抽出・分類]
    ↓
[~/dotfiles/.claude/skills/guideline-*/SKILL.md に追記]
    ↓
[次回実装時にClaude Codeが自動参照]
    ↓
[同じ指摘を繰り返さない]
```
